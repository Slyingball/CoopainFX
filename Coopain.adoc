```sql
SELECT t.idNational, t.nom,SUM(l.stockRestant)AS stock_total
FROM Taureau t
LEFT JOIN Lot lON t.idNational= l.idNationalTaureau
GROUP BY t.idNational, t.nom
ORDER BY stock_totalASC;
```

### **Question 2.3 (Analyse des risques et DICP) :** En cas de perte ou de vol de l’ordiphone d’un inséminateur, évaluez les risques pesant sur la base de données locale `SQLite` au regard des critères de **Confidentialité** et d'**Intégrité** (matrice DICP). Proposez au moins une solution technique ou organisationnelle (ex: MDM, chiffrement) de protection des données sur cet appareil mobile.

- **Confidentialité** : Les données de la base SQLite locale (Adhérents, adresses) sont stockées en clair. N'importe qui peut lire le fichier base de données.
- **Intégrité** : Le voleur pourrait modifier les kilomètres ou les données des inséminations sur l'appareil avant toute synchronisation.
- **Solution** : Chiffrement du fichier de base de données SQLite (ex: avec la bibliothèque SQLCipher) + Utilisation d'un MDM (Mobile Device Management) pour permettre l'effacement à distance des données de l'appareil.

### **Question 2.4 (Vulnérabilité OWASP) :** L’analyse du code métier (Document 10) indique la présence de l’attribut `private String password;` au sein de la classe `Inseminateur`. Expliquez en quoi le stockage ou la transmission de cette donnée en clair représente une faille critique. Que recommandez-vous de mettre en place pour sécuriser la phase d’authentification ?

- **Faille** : Le mot de passe stocké (`String password`) transite en clair dans la mémoire de l'application et sur le réseau. C'est une vulnérabilité critique d'Identification et d'Authentification (A07:2021).
- **Recommandation** : Ne jamais stocker de mots de passe en clair. Utiliser une API sécurisée en HTTPS et ne conserver sur le téléphone qu'un **Token d'authentification** (ex: JWT) temporaire.

### **Question 3.3 (Cybersécurité des API) :** La synchronisation des tournées entre l’application `MobiSemin` et le serveur `LogiSemin` se fait via le réseau Internet. Décrivez une attaque possible visant à intercepter ou altérer ces données en transit (Man-in-the-Middle) et indiquez le protocole standard à exiger côté serveur pour chiffrer ces échanges.

- **L'attaque** : Sur un réseau non sécurisé (ex: wifi public), un pirate peut intercepter silencieusement les données qui circulent entre le smartphone MobiSemin et le serveur LogiSemin. S'il n'y a pas de chiffrement, il peut lire ou modifier ces données (Man-in-the-Middle).
